# Glossary

**2FA** - Two-Factor Authentication. A second verification step beyond a password.

**Air-gapped** - A machine with no network connections whatsoever.

**DoH / DoT / DoQ** - DNS over HTTPS / TLS / QUIC. Protocols for encrypting DNS queries.

**E2EE** - End-to-End Encryption. Only communicating parties can read messages; the server cannot.

**EXIF** - Metadata in image files, often including GPS coordinates, device model, and timestamp.

**Fingerprinting** - Identifying a user by combining browser/device attributes without cookies.

**FIDO2** - Standard for hardware-based authentication (passkeys, security keys). Phishing-resistant.

**KYC** - Know Your Customer. Identity verification requirements for financial services.

**LUKS** - Linux Unified Key Setup. Standard for full disk encryption on Linux.

**Metadata** - Data about data: who, when, where - not the content itself.

**Monero (XMR)** - Privacy-focused cryptocurrency with hidden sender, recipient, and amounts.

**OPSEC** - Operational Security. Practices to prevent adversaries gathering information about you.

**OSINT** - Open-Source Intelligence. Gathering information from publicly available sources.

**Passkey** - FIDO2-based credential replacing passwords using public-key cryptography.

**PGP** - Pretty Good Privacy. Email encryption standard using public/private key pairs.

**Pseudonymity** - Acting under a consistent identity that isn't your real name.

**SIM Swapping** - Attack where an attacker transfers your phone number to their SIM to intercept SMS 2FA.

**SNI** - Server Name Indication. TLS extension revealing the hostname, visible to network observers.

**TOTP** - Time-based One-Time Password. 6-digit codes expiring every 30 seconds.

**Threat Model** - Structured analysis of what you protect, from whom, and how.

**VPN** - Virtual Private Network. Encrypts traffic and routes it through a server, hiding it from your ISP.

**Zero-Knowledge** - Service design where the provider cannot access your data - only you have the keys.
