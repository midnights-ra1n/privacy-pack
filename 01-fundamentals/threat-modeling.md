# Threat Modeling
> 🟢 Start here before anything else.

## The 5 Questions
1. **What** do you want to protect?
2. **Who** are you protecting it from?
3. **How likely** is the threat?
4. **How bad** are the consequences?
5. **How much effort** are you willing to invest?

## Example Profiles
| Profile | Adversaries | Starting Point |
|---------|------------|----------------|
| 🟢 Average user | Advertisers, data brokers | Password manager, 2FA, Signal |
| 🟡 Privacy-conscious | Corporate surveillance, ISP | VPN, ProtonMail, Linux |
| 🔴 Journalist/Activist | Nation-states, law enforcement | Tails, Tor, compartmentalization |
| ⚫ High-risk | Sophisticated state actors | Qubes OS, air-gapped machines |

## Common Mistakes
- Adopting maximum security for a low-risk situation
- Focusing on software, ignoring physical security
- Thinking one tool solves everything
- "I have nothing to hide" : privacy is about control, not hiding wrongdoing
